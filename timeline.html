Proposed

Exact days will be aligned with the official ICSE 2027 Competition Track schedule.

The accepted proposal specifies milestone months rather than final deadlines. Dates below should be used for planning, not as binding submission cutoffs.

Proposal milestones

Planned competition sequence

  1. July 2026Current preparation phase
    In progress

    Public website and starter kit preparation

    Publish the competition overview, stabilize the Agent contract, harden evaluator infrastructure, and prepare participant documentation.

    • Multipage website preview
    • Agent API design
    • OBS and local validator integration
    • Data audit and metadata normalization
  2. August 2026Proposed
    Planned

    Development set and baselines released

    Release public development cases, conformance examples, evaluator documentation, and reproducible baseline Agents.

  3. September 2026Proposed
    Planned

    Public validation leaderboard opens

    Accept versioned Agent entries, run public validation cases, and return bounded diagnostic feedback.

  4. November 2026Proposed
    Planned

    Final submission freeze

    Freeze each team's selected Agent artifact, the interface version, evaluator, case manifests, and resource policy for hidden evaluation.

  5. December 2026Proposed
    Planned

    Final results and participant reports

    Complete hidden evaluation, audit finalist repairs, publish aggregate results, and collect eligible participant reports.

  6. ICSE 2027Official conference schedule TBA
    Conference

    Competition session and results presentation

    Present the benchmark, leading systems, common failure modes, and final competition outcomes to the ICSE community.

Release gates

What must be ready before public validation

Gate AAgent contract frozen

Entrypoint, input/output schemas, packaging, conformance tests, and failure semantics are versioned.

Gate BEvaluation policy frozen

Metrics, budgets, allowed APIs, network policy, tie-breakers, and submission limits are published.

Gate CDataset release audited

Splits, licensing, checksums, immutable paths, and near-duplicate grouping are verified.

Gate DEnd-to-end path tested

Example Agent submission, case execution, OBS validation, structured result, and leaderboard update all succeed.

Final evaluation

Freeze first, then run hidden cases

1Select final entry
2Freeze versions
3Run hidden set
4Audit results
5Publish ranking

The final hidden evaluation is expected to finish within one day under the initial proposal target. The exact queue, rerun window, and result publication time will be announced with final deadlines.
